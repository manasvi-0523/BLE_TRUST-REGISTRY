# Code of Conduct

## Our Pledge

We, as members, contributors, and maintainers of the BLE Trust Registry project, pledge to make participation in this project a respectful and collaborative experience for everyone, regardless of academic background, experience level, gender, nationality, or personal identity.

This is an academic research project at Don Bosco Institute of Technology, Bengaluru. We hold ourselves to the professional and ethical standards expected in academic and open-source communities.

## Our Standards

**Expected behavior:**

- Treat all contributors and users with respect and professionalism
- Give and receive constructive feedback graciously
- Acknowledge the contributions and ideas of others
- Focus discussions on technical merit and project goals
- Be patient with contributors who are learning
- Use clear, inclusive language in all communication

**Unacceptable behavior:**

- Harassment, intimidation, or discrimination of any kind
- Dismissive or disrespectful responses to questions or contributions
- Plagiarism, presenting others' work as your own
- Misuse of the project for unauthorized surveillance or monitoring
- Using project tools or data on devices/networks without explicit permission
- Sharing or publishing private data (MAC addresses, device identifiers) collected using this tool

## Academic Integrity

This project is built in an academic context. Contributors are expected to:

- Properly attribute any algorithms, code, or ideas borrowed from external sources
- Not submit contributions to this project as work for academic assignments without disclosure
- Cite this project appropriately in any academic work that builds on it

## Responsible Use

BLE scanning involves collecting radio frequency data from nearby devices. Contributors and users must:

- Only run scans on networks and devices they own or have explicit written permission to monitor
- Not use this tool to track individuals without their knowledge and consent
- Comply with local laws and regulations regarding radio monitoring and data collection
- Treat any device data collected as sensitive; do not share, publish, or expose MAC addresses or behavioral profiles of real devices

## Reporting Issues

If you observe a violation of this Code of Conduct, or have concerns about how the project or its tools are being used:

- For conduct issues within the project: contact the maintainers via the GitHub repository
- For security vulnerabilities: do not open a public issue; email the maintainers directly (see CONTRIBUTING.md)
- For urgent matters: contact the faculty supervisor, Dr. Sheeba, at Don Bosco Institute of Technology, Bengaluru

Reports will be reviewed by the core team (Team NEXUS ONLINE). We are committed to handling reports with discretion and responding within a reasonable timeframe.

## Enforcement

Project maintainers are responsible for clarifying and enforcing this Code of Conduct. Responses to violations may include:

1. **Correction** - A private note explaining the issue and expected behavior
2. **Warning** - A formal warning with conditions for continued participation
3. **Temporary suspension** - Temporary removal of contribution privileges
4. **Permanent removal** - Permanent removal from the project for serious or repeated violations

Maintainers who do not uphold this Code of Conduct in good faith may face consequences determined by the project leadership and faculty oversight.

## Scope

This Code of Conduct applies to all project spaces including the GitHub repository, issues, pull requests, and any communication channels used by the project team. It also applies when an individual is representing the project in public spaces.
